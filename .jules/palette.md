## 2024-06-19 - Accessible Custom Form Elements
**Learning:** Custom interactive elements (like the theme color picker circles in `NoticesDashboard.tsx`) inherently lack the native accessibility semantics of standard inputs (like `<select>` or `<input type="color">`). Without explicit mapping of visual elements to human-readable names, screen reader users only hear "button", and keyboard users have no context for the selected state.
**Action:** When building or modifying custom UI controls (especially color pickers or icon sets), map visual states to human-readable text. Apply `role="group"` with an `aria-label` for the container, and use `aria-label` (for screen readers), `title` (for mouse hover tooltips), and `aria-pressed` (for state feedback) on the individual interactive elements to ensure parity across all interaction modalities.
